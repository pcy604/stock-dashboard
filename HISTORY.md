# 📜 개발 히스토리 — 시그널 트레이딩 시스템

> 한·미 주식 시그널 스크리너에서 출발해, **백테스트로 스스로를 반증하며** 진화한
> 퀀트 트레이딩 의사결정 시스템. 핵심은 기능의 양이 아니라, **데이터가 가설을 뒤집을
> 때마다 전략을 바꾼 의사결정의 궤적**이다.

기술 스택: Python · Streamlit · FinanceDataReader · yfinance · 네이버금융 스크래핑 ·
FRED API · pandas/numpy · Plotly · GitHub + Streamlit Cloud 배포

## 4기 — 데이터에 의한 전략 대전환 (2026.06.20)

이날이 이 프로젝트의 **분기점**이다. 백테스트가 우리의 핵심 가설들을 차례로 반증했고,
그때마다 전략을 바꿨다.

| 발견 (백테스트) | 반증된 가설 | 바뀐 전략 |
|------|------|------|
| **레짐 백테스트** — 추세장에서 고정보유가 가격 트레일링스톱을 **+16%p 이김** | "똑똑한 트레일링 청산이 수익을 지킨다" | 가격 기반 청산 폐기. 트레일링은 노이즈에 당함 |
| **연도별 백테스트(2021~2025)** — 위닝 스코어 등급의 변별력이 **비일관**, 2026 호조는 운 | "좋은 점수 = 높은 수익" | **종목 선정 매달리기 중단** |
| 위 둘의 종합 | "무엇을 사느냐가 핵심" | **"어떻게 살아남느냐(매도·리스크 관리)"로 무게중심 이동** (멍거 inversion) |

**실행:**
- 통합 1단계 — 중복 탭(추천포트) 폐기, 자동추천에 흡수
- **매도 신호 엔진**(`sell_signals`) — 방어손절·분할익절·시간매도. *사후적 이평 이탈 룰은 배제*,
  미리 정한 사전 규칙으로만 매도

**4기 성과(진행 중):** "더 좋은 신호"를 좇던 시스템이, **"작게 잃고 살아남는"** 시스템으로 전환.

## 이 프로젝트가 보여주는 것

1. **반증 가능성** — 자기 가설을 백테스트로 깨고, 깨지면 인정하고 바꿨다.
2. **정직한 검증** — 비용·생존편향·OOS·레짐까지 차감한 뒤에야 결론을 냈다.
3. **운영까지** — 로컬 스크립트가 아니라 클라우드 배포·자동화·포워드 검증까지.
4. **철학의 진화** — "이기는 법"에서 "안 망하는 법"으로. 데이터가 시킨 성숙.

> "Invert, always invert." — Charlie Munger
